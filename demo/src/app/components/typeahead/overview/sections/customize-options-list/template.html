As said above, the options list can contain any kind of item, and this is handy since selecting an option will commit the actual item to the data model, allowing to store complex objects.

But the requirement is to be able to display those options, be it in the list of the popup or inside the input field.

Simple representation with a string

The option item can be displayed from a string. In this case, use the property resultFormatter to pass a function which takes the option item (whose model is up to you) as input and returns a string as output.

Complex representation with a template

Sometimes displaying a simple string is not enough. In that case, you can use a template by using the property resultFormatter, which expects a template reference.

The template receives in its context:

  • the option item, as result
  • the input field content (the search term), as term
