Editing the search term

The classic usage of the Typeahead is to use the input to edit the search term, and trigger a search process which will spawn a new options list.

Apart from creating an Oservable based on the input event of the input, the Typeahead does nothing else specific, so to edit the search term just use this input as any other input.

Navigating through the options

Once the popup is open, you can navigate through the options using the keyboard arrow up and down.

Currently activated option will be highlighted so you know where you are.

You can also hover the options with the mouse to activate a different option. Starting keyboard navigation again will always start from the currently activated option.

Selecting an option

To actually select an option (commit it to the data model), you can either directly click or tap on it, or use the enter key on the currently activated option.

Dismissing the popup

Dismissing the popup consists in closing the popup without selecting an option.

This can be done by:

  • spawning an empty options list from the Observable (will always work)
  • pressing the escape key (requires option autoclose to be any value other than falsefalse)
  • clicking inside the widget (requires option autoclose to be true or 'inside')
  • clicking outside the widget (requires option autoclose to be true or 'outside')
